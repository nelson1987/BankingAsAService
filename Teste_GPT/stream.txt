O c#7.0 consegue realizar stream como a Netflix?
C# 7.0 é uma versão da linguagem de programação C# que possui recursos para facilitar a escrita de código, mas não possui recursos específicos para streaming de vídeo, como a Netflix. No entanto, é possível usar a biblioteca de streaming de vídeo do C# para desenvolver aplicativos que transmitem vídeo, mas é preciso escrever código adicional para implementar as funcionalidades de streaming desejadas.

Quais bibliotecas poderiam ser utilizadas?
Existem várias bibliotecas que podem ser usadas para desenvolver aplicativos de streaming de vídeo com C#, algumas delas incluem:

MediaElement: Essa é uma classe de elemento de mídia embutida no WPF (Windows Presentation Foundation) e no XAML (eXtensible Application Markup Language) que pode ser usada para reproduzir vídeos em aplicativos desktop.

FFmpeg: É uma biblioteca de software livre que pode ser usada para codificar, decodificar, transcodificar e filtrar arquivos de áudio e vídeo. A biblioteca é compatível com vários formatos de arquivo e pode ser usada para desenvolver aplicativos de streaming de vídeo.

libVLC: É uma biblioteca de código aberto que implementa o protocolo de streaming de vídeo VLC. Ela pode ser usada para desenvolver aplicativos de streaming de vídeo que reproduzem arquivos de vídeo em diferentes formatos.

NAudio: É uma biblioteca de áudio para C# e VB.NET que fornece uma interface para acessar dispositivos de áudio e realizar operações de áudio. Ela pode ser usada para desenvolver aplicativos de streaming de áudio e vídeo.

WebRTC: É uma tecnologia de comunicação em tempo real para navegadores e dispositivos móveis que permite a transmissão de áudio e vídeo entre dispositivos. Ele tem suporte a C# e pode ser usado para desenvolver aplicativos de streaming de vídeo.

Essas são apenas algumas das bibliotecas disponíveis, existem muitas outras bibliotecas e ferramentas disponíveis para desenvolver aplicativos de streaming de vídeo com C#.
